Perche OOP in PHP

La programmazione a oggetti (OOP) aiuta a organizzare progetti grandi: separa responsabilita, rende il codice riusabile e facilita manutenzione e test.

Idea chiave: un oggetto combina dati (proprieta) e comportamenti (metodi).

Classi e Oggetti

<?php
class Studente {
    public string $nome;

    public function saluta(): string {
        return "Ciao, sono " . $this->nome;
    }
}

$studente = new Studente();
$studente->nome = "Giulia";

echo $studente->saluta();
?>

La classe e un modello. L'oggetto e una istanza concreta creata con new.

Costruttore e Inizializzazione

<?php
class Corso {
    public function __construct(
        public string $titolo,
        public int $anno
    ) {}
}

$corso = new Corso("Web Design 2", 2026);
echo $corso->titolo;
?>

Il costruttore __construct viene eseguito quando crei un oggetto.

Visibilita: public, protected, private

Keyword Accessibile da Uso tipico
public Ovunque API della classe
protected Classe e sottoclassi Dettagli condivisi in ereditarieta
private Solo nella classe stessa Incapsulamento dati sensibili 
<?php
class Account {
    private float $saldo = 0;

    public function deposita(float $importo): void {
        if ($importo > 0) {
            $this->saldo += $importo;
        }
    }

    public function getSaldo(): float {
        return $this->saldo;
    }
}
?>

Ereditarieta e Polimorfismo

<?php
class Utente {
    public function ruolo(): string {
        return "utente";
    }
}

class Admin extends Utente {
    public function ruolo(): string {
        return "admin";
    }
}

function stampaRuolo(Utente $u): void {
    echo $u->ruolo();
}

stampaRuolo(new Utente());
stampaRuolo(new Admin());
?>

Con il polimorfismo usi lo stesso contratto su oggetti diversi.

Interfacce e Trait

Interfacce

<?php
interface Renderable {
    public function render(): string;
}

class Card implements Renderable {
    public function render(): string {
        return "<div>Card</div>";
    }
}
?>

Trait

<?php
trait Loggable {
    public function log(string $msg): void {
        echo "[LOG] " . $msg;
    }
}

class Servizio {
    use Loggable;
}
?>

Metodi e Proprieta Statiche

<?php
class Config {
    public static string $env = "production";

    public static function isProd(): bool {
        return self::$env === "production";
    }
}

echo Config::isProd() ? "Prod" : "Dev";
?>

Gli elementi statici appartengono alla classe, non alla singola istanza.

Confronto OOP: JavaScript vs PHP

JavaScript
class Studente {
  constructor(nome) {
    this.nome = nome;
  }

  saluta() {
    return `Ciao ${this.nome}`;
  }
}
PHP
<?php
class Studente {
    public function __construct(public string $nome) {}

    public function saluta(): string {
        return "Ciao " . $this->nome;
    }
}
?>
Nota didattica: in JavaScript gli studenti hanno gia visto classi lato browser; in PHP le applichiamo a logica server, modelli dati e servizi.

Best Practice OOP

  • Una classe, una responsabilita chiara.
  • Preferisci composizione a ereditarieta quando possibile.
  • Tipizza parametri e ritorni (strict typing quando adatto).
  • Evita classi monolitiche con troppi compiti.
  • Struttura il progetto con cartelle dedicate (Models, Services, Controllers).
