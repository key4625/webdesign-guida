Cos'è GSAP?

GSAP (GreenSock Animation Platform) è lo standard industriale per le animazioni web JavaScript. È robusto, veloce e risolve automaticamente le incompatibilità tra browser.

Perché usarlo? Rispetto alle animazioni CSS, GSAP offre un controllo molto più preciso su sequenze complesse (Timeline), performance migliori su animazioni pesanti e plugin potenti come ScrollTrigger.

Installazione

Il modo più semplice per iniziare è includere la CDN nel tuo file HTML, prima della chiusura del body.

<!-- GSAP Core -->
<script src="https://cdnjs.cloudflare.com/ajax/libs/gsap/3.12.5/gsap.min.js"></script>

<!-- ScrollTrigger (Opzionale, per animazioni allo scroll) -->
<script src="https://cdnjs.cloudflare.com/ajax/libs/gsap/3.12.5/ScrollTrigger.min.js"></script>

Interpolazioni (Tweens)

Il cuore di GSAP è il "Tween" (interpolazione). Un tween modifica le proprietà di un elemento nel tempo.

gsap.to()

Anima un elemento dallo stato attuale verso i valori definiti.

// Sposta l'elemento con classe .box di 100px a destra in 1 secondo
gsap.to(".box", { x: 100, duration: 1 });

gsap.from()

Anima un elemento da valori definiti verso lo stato attuale (ottimo per entrate in scena).

// L'elemento appare venendo dall'alto con opacità 0
gsap.from(".box", { y: -100, opacity: 0, duration: 1 });

gsap.fromTo()

Definisce sia lo stato di partenza che quello di arrivo.

gsap.fromTo(".box", 
    { x: 0, opacity: 0 }, // from
    { x: 200, opacity: 1, duration: 2 } // to
);

Proprietà Comuni

GSAP usa proprietà trasformate per massimizzare le performance. Ecco le più usate:

Proprietà GSAP Corrispondente CSS Descrizione
x, y transform: translateX(), translateY() Spostamento orizzontale/verticale.
duration animation-duration Durata in secondi (default: 0.5).
rotation transform: rotate() Rotazione in gradi.
scale transform: scale() Scala dell'elemento.
opacity opacity Trasparenza (0 a 1).
delay animation-delay Ritardo prima dell'inizio.
stagger - Tempo tra l'inizio dell'animazione di ogni elemento in un gruppo. 
// Esempio con Stagger (animazione a cascata)
gsap.to(".box", { 
    y: 50, 
    opacity: 0, 
    duration: 0.5, 
    stagger: 0.2 // 0.2 secondi tra ogni elemento
});

Easing

L'Easing determina la "personalità" dell'animazione (accelerazione e decelerazione). Il default è "power1.out".

  • "none": Lineare (velocità costante).
  • "power1", "power2", "power3", "power4": Curve di accelerazione standard.
  • "back": Supera leggermente il target e torna indietro.
  • "elastic": Effetto elastico.
  • "bounce": Effetto rimbalzo.
gsap.to(".box", { 
    x: 300, 
    duration: 1.5, 
    ease: "bounce.out" 
});

Staggers

Gli Staggers (sfalsamenti) sono una delle funzionalità più amate di GSAP. Permettono di animare un gruppo di elementi con lo stesso codice, aggiungendo un piccolo ritardo tra l'inizio dell'animazione di ciascuno.

Demo Live: 
gsap.to(".stagger-box", {
    y: -20,
    duration: 0.5,
    stagger: 0.1, // 0.1 secondi di ritardo tra ogni elemento
    repeat: -1,
    yoyo: true
});

Puoi anche passare un oggetto alla proprietà stagger per un controllo più avanzato, utile per griglie o animazioni complesse:

stagger: {
    each: 0.1,
    from: "center", // Inizia dal centro
    grid: "auto",
    ease: "power2.inOut"
}

SVG e GSAP

GSAP e SVG sono una combinazione perfetta. A differenza dei div, gli SVG permettono di disegnare forme complesse che scalano all'infinito senza perdere qualità. L'SVG (Scalable Vector Graphics) è un formato XML che descrive grafica vettoriale: le forme sono definite matematicamente, quindi non perdono mai qualità ridimensionando.

1. Creare e incorporare SVG

Puoi creare SVG con software come Illustrator, Inkscape o Figma. Una volta disegnato:

  1. Esporta come codice SVG (non immagine PNG/JPG).
  2. Copia il codice (che inizia con <svg>) e incollalo direttamente nel tuo HTML.
  3. Assegna class o id agli elementi interni che vuoi animare (es. class="my-circle").
Importante: Per poter animare gli elementi interni con GSAP, l'SVG deve essere inline nel codice HTML, non un tag <img src="file.svg">.

Forme SVG Principali

Ogni forma SVG ha attributi geometrici specifici che ne definiscono posizione e dimensioni. Tutti questi attributi sono animabili con GSAP tramite attr: {}.

Demo Live — tutte le forme:

<rect> x, y, width, height, rx, ry Rettangolo. rx/ry arrotondano gli angoli.
<ellipse> cx, cy, rx, ry Ellisse. Due raggi separati per orizzontale e verticale.
<line> x1, y1, x2, y2 Linea retta tra due punti.

Cosa si può animare con GSAP

GSAP suddivide le proprietà animabili degli SVG in tre categorie:

A) Proprietà CSS (dirette, come su un div)

Si passano direttamente nell'oggetto tween, senza attr:

gsap.to(".mia-forma", {
    opacity: 0,        // trasparenza
    fill: "#ff0000",   // colore riempimento
    stroke: "#000",    // colore bordo
    strokeWidth: 5,    // in GSAP si usa camelCase
    x: 100,            // translateX (transform)
    y: -50,            // translateY (transform)
    rotation: 45,      // rotate (transform)
    scale: 1.5,        // scale (transform)
    transformOrigin: "center center",
    duration: 1
});

B) Attributi geometrici (con attr: {})

Per cambiare la forma stessa (raggio, coordinate, dimensioni) si usa l'oggetto attr:

// Anima il raggio di un cerchio
gsap.to(".svg-circle", {
    attr: { r: 20 },
    duration: 1
});

// Anima larghezza e angoli di un rettangolo
gsap.to(".svg-rect", {
    attr: { width: 150, rx: 40 },
    duration: 1
});

// Anima i raggi di un'ellisse
gsap.to(".svg-ellipse", {
    attr: { rx: 80, ry: 15 },
    duration: 1
});

// Anima i punti di una linea
gsap.to(".svg-line", {
    attr: { x2: 500, y2: 20 },
    duration: 1
});

C) Effetto "traccia che si disegna" con stroke-dashoffset

Questa tecnica è molto popolare: impostando stroke-dasharray uguale alla lunghezza del tracciato e animando stroke-dashoffset da quel valore a 0, la linea sembra disegnarsi progressivamente.

// Prima misura la lunghezza del path con JS
const path = document.querySelector(".mio-path");
const length = path.getTotalLength();

// Imposta lo stato iniziale (linea invisibile)
gsap.set(path, {
    attr: {
        strokeDasharray: length,
        strokeDashoffset: length
    }
});

// Anima fino a 0: la linea "si disegna"
gsap.to(path, {
    attr: { strokeDashoffset: 0 },
    duration: 2,
    ease: "power2.inOut"
});

Testi con GSAP

GSAP può animare qualsiasi elemento HTML che contenga testo — titoli, paragrafi, span — esattamente con le stesse proprietà usate per i div. Non è necessario che il testo sia in un SVG.

1. Animare un intero blocco di testo

Il caso più semplice: muovere, sfumare o scalare un titolo come se fosse una scatola qualunque.

// Entrata dall'alto con fade-in
gsap.from("h1", {
    y: -60,
    opacity: 0,
    duration: 1,
    ease: "power3.out"
});

// Entrata con leggero zoom
gsap.from(".subtitle", {
    scale: 0.5,
    opacity: 0,
    duration: 0.8,
    delay: 0.4,
    ease: "back.out(1.7)"
});

2. Animare lettera per lettera (splitting manuale)

Dividendo il testo in tanti <span> (uno per lettera), GSAP può animare ogni carattere singolarmente con lo stagger. Non serve nessun plugin: lo si fa con poche righe di JavaScript.

Demo Live:
GSAP

// 1. Suddividi il testo in caratteri
const el = document.querySelector("#titolo");
const chars = el.textContent.split("");
el.innerHTML = chars
    .map(c => `<span class="char">${c === " " ? "&nbsp;" : c}</span>`)
    .join("");

// 2. Anima tutti gli span con stagger
gsap.from(".char", {
    y: 60,
    opacity: 0,
    duration: 0.5,
    stagger: 0.05,  // 50ms tra ogni lettera
    ease: "back.out(1.7)"
});

4. TextPlugin — animare il contenuto del testo

Il TextPlugin (gratuito, incluso in GSAP) permette di animare il contenuto di un elemento testo: le parole appaiono carattere per carattere come in un effetto macchina da scrivere.

// Registra il plugin (una sola volta)
gsap.registerPlugin(TextPlugin);

gsap.to("#titolo", {
    duration: 2,
    text: "Ciao, sono GSAP!", // Scrive questo testo carattere per carattere
    ease: "none"
});

// Con delimitatore personalizzato (parola per parola invece di lettera per lettera)
gsap.to("#paragrafo", {
    duration: 3,
    text: {
        value: "Le parole appaiono una alla volta.",
        delimiter: " " // Spezza per spazi (parole)
    },
    ease: "none"
});
CDN TextPlugin: Aggiungi questo script dopo il core GSAP per usarlo: 
<script src="https://cdnjs.cloudflare.com/ajax/libs/gsap/3.12.5/TextPlugin.min.js"></script>

5. Combinare testi e Timeline

L'abbinamento più potente è mettere le animazioni di testo in una Timeline, così le sequenze narrative diventano facili da gestire:

const tl = gsap.timeline();

tl.from(".hero-title", { y: -80, opacity: 0, duration: 0.8, ease: "power3.out" })
  .from(".hero-subtitle", { y: 30, opacity: 0, duration: 0.6, ease: "power2.out" }, "-=0.3")
  .from(".hero-cta", { scale: 0, opacity: 0, duration: 0.4, ease: "back.out(2)" }, "-=0.2");

Timeline

Le Timeline permettono di concatenare animazioni in sequenza senza dover calcolare manualmente i ritardi (delays).

let tl = gsap.timeline();

// Le animazioni avvengono una dopo l'altra
tl.to(".box1", { x: 100, duration: 1 })
  .to(".box2", { y: 50, duration: 0.5 }) // Parte quando finisce la precedente
  .to(".box3", { rotation: 360, duration: 1 }, "-=0.5"); // Parte 0.5 secondi PRIMA della fine della precedente

ScrollTrigger

ScrollTrigger è il plugin più famoso di GSAP: permette di agganciare le animazioni allo scroll della pagina.

È necessario registrare il plugin prima dell'uso:

gsap.registerPlugin(ScrollTrigger);

gsap.to(".box", {
    x: 400,
    rotation: 360,
    scrollTrigger: {
        trigger: ".box", // L'elemento che fa scattare l'animazione
        start: "top center", // Quando la cima dell'elemento raggiunge il centro della finestra
        end: "bottom 100px",
        scrub: true, // L'animazione segue lo scroll (se false, parte e basta)
        markers: true // Mostra marcatori per debug (ricordati di toglierlo in produzione!)
    }
});
