Primi Passi

Iniziare con React

Per usare React in modo professionale, non basta un singolo file HTML. Abbiamo bisogno di un ambiente di sviluppo completo.

Prerequisito: Assicurati di avere Node.js installato sul tuo computer.

1. Creare il Progetto (Create React App)

Il modo più classico per iniziare (e quello suggerito da molte guide introduttive) è usare il comando ufficiale create-react-app. Questo comando prepara tutto l'ambiente automaticamente.

Apri il tuo terminale (Prompt dei comandi o PowerShell) e digita:

npx create-react-app my-react-app

Nota: my-react-app è il nome della cartella che verrà creata. Puoi chiamarla come vuoi.

Una volta finito (ci vorrà qualche minuto), entra nella cartella e avvia il progetto:

cd my-react-app
npm start

Se tutto è andato bene, si aprirà automaticamente il browser all'indirizzo http://localhost:3000 e vedrai il logo di React girare!

2. Struttura del Progetto

Apri la cartella del progetto con VS Code. Vedrai diversi file, ma quelli importanti sono pochi:

  • node_modules/: Contiene tutte le librerie (le dipendenze) scaricate. È una cartella pesantissima, non toccarla mai!
  • public/: Contiene i file statici come index.html e le immagini.
  • src/: Qui è dove lavorerai! Contiene tutto il codice sorgente (JavaScript/JSX e CSS).
    • App.js: Il componente principale della tua applicazione.
    • index.js: Il punto di ingresso che "monta" React nella pagina HTML.
    • App.css: Lo stile del componente App.
  • package.json: Il file di "identità" del progetto. Elenca le librerie installate e i comandi disponibili.

3. La Tua Prima Modifica

Ora modifichiamo la pagina di benvenuto di React per farla nostra.

  1. Tieni il server in esecuzione (non chiudere il terminale dove hai scritto npm start).
  2. Vai nella cartella src e apri il file App.js.
  3. Cerca il codice dentro il return (...). Sembra HTML.
  4. Cancella tutto quello che c'è dentro il <div className="App"> e scrivi un titolo:

Il tuo codice dovrebbe apparire così:

function App() {
  return (
    

      
Ciao Mondo!

      
Questa è la mia prima applicazione React.

    

  );
}

export default App;

Salva il file (`Ctrl+S`). Guarda il browser: si aggiornerà magicamente da solo senza dover ricaricare la pagina (Hot Reloading)!

Teoria Essenziale

Pensare in React

Cos'è React e il JSX?

Dimentica di scrivere HTML in un file e JS in un altro. In React scriviamo JSX (JavaScript XML).

Il JSX assomiglia all'HTML, ma in realtà è JavaScript travestito! Ci permette di descrivere l'interfaccia utente direttamente dentro la logica del nostro codice.

Regole d'Oro del JSX
  • Classi CSS: In HTML usi class, ma in JSX devi usare className (perché "class" è una parola riservata in JS).
  • Tag Chiusi: Tutti i tag devono essere chiusi. <br> diventa <br />.
  • IlPortale JS: Le parentesi graffe { } sono magiche: ti permettono di inserire variabili o calcoli JavaScript direttamente nel markup.

I Componenti Funzionali

Un componente in React non è nient'altro che una Funzione JavaScript che restituisce del codice JSX (la UI).

// Definisco il componente (Iniziale Maiuscola!)
function Header() {
    const titolo = "Benvenuti!";
    return (
        

             
{titolo}

        

    );
}

Props (Dati in sola lettura)

Le Props sono come i parametri di una funzione. Permettono al componente "genitore" di passare dati al componente "figlio" per configurarlo.

  • Immagina le Props come gli "ingredienti" che passi alla funzione per cucinare la pagina.
  • Nota: Le Props sono di sola lettura. Il figlio non può modificarle!

State (La memoria del componente)

A volte un componente ha bisogno di "ricordare" qualcosa, ad esempio se un bottone è stato cliccato o il valore di un contatore. Questa memoria si chiama Stato.

Usiamo l'Hook useState:

import { useState } from 'react';

function Contatore() {
    // [valoreAttuale, funzionePerCambiarlo] = useState(valoreIniziale)
    const [contatore, setContatore] = useState(0);

    return (

    );
}
