Introduzione

Le applicazioni web moderne comunicano costantemente con i server per scambiare dati. Questo avviene tramite API (Application Programming Interface), spesso seguendo lo standard REST.

Cosa significa CRUD?

CRUD è l'acronimo delle quattro operazioni fondamentali per la gestione persistente dei dati:

  • Create (Creazione) - Solitamente metodo HTTP POST
  • Read (Lettura) - Solitamente metodo HTTP GET
  • Update (Aggiornamento) - Solitamente metodo HTTP PUT o PATCH
  • Delete (Cancellazione) - Solitamente metodo HTTP DELETE
Nota: In JavaScript, utilizziamo la Fetch API o librerie come Axios per eseguire queste operazioni in modo asincrono.

Questa API permette di gestire utenti e post (articoli). Per le operazioni di modifica (Create, Update, Delete) è necessaria l'autenticazione tramite Token JWT.

Autenticazione

Prima di poter modificare i dati, dobbiamo ottenere un token di accesso effettuando il login.

Login

Inviamo le credenziali (email e password) all'endpoint /login.

const API_URL = "https://micheleabamc.netsons.org/wd2/api";

async function login(email, password) {
    try {
        const response = await fetch(`${API_URL}/login`, {
            method: "POST",
            headers: {
                "Content-Type": "application/json"
            },
            body: JSON.stringify({ email, password })
        });

        const data = await response.json();
        
        if (response.ok) {
            console.log("Login effettuato con successo!");
            // Salviamo il token per le chiamate successive
            localStorage.setItem("token", data.token);
            return data.token;
        } else {
            console.error("Errore login:", data.message);
        }
    } catch (error) {
        console.error("Errore di rete:", error);
    }
}

Lettura Dati (GET)

Per recuperare i dati, utilizziamo il metodo GET. Questo endpoint è solitamente pubblico.

async function getPosts() {
    try {
        const response = await fetch(`${API_URL}/posts`);
        const posts = await response.json();
        
        console.log("Post recuperati:", posts);
        
        // Esempio: Visualizzazione nella pagina
        const container = document.getElementById("posts-container");
        container.innerHTML = posts.map(post => `
            <div class="post">
                <h3>${post.title}</h3>
                <p>${post.content}</p>
            </div>
        `).join('');
        
    } catch (error) {
        console.error("Errore nel recupero dei post:", error);
    }
}

Creazione (POST)

Per creare un nuovo post, usiamo il metodo POST. È necessario includere il token nel header Authorization.

Se devi inviare anche file (come immagini), è meglio usare l'oggetto FormData invece di JSON.

async function createPost(title, content, fileInput) {
    const token = localStorage.getItem("token");
    if (!token) {
        console.error("Utente non autenticato!");
        return;
    }

    // Usiamo FormData per gestire anche il caricamento file
    const formData = new FormData();
    formData.append('title', title);
    formData.append('content', content);
    formData.append('category', 'Generale');
    if (fileInput && fileInput.files[0]) {
        formData.append('image', fileInput.files[0]);
    }

    try {
        const response = await fetch(`${API_URL}/posts`, {
            method: "POST",
            headers: {
                // Non impostare 'Content-Type' con FormData, il browser lo fa automaticamente
                "Authorization": `Bearer ${token}`
            },
            body: formData
        });

        const result = await response.json();
        console.log("Risultato:", result);

    } catch (error) {
        console.error("Errore creazione post:", error);
    }
}

Aggiornamento (PUT)

Per modificare una risorsa esistente, usiamo PUT specificando l'ID della risorsa nell'URL.

async function updatePost(id, title, content) {
    const token = localStorage.getItem("token");
    
    // Nota: Se usiamo PUT per inviare solo JSON (senza file)
    const data = new URLSearchParams();
    data.append('title', title);
    data.append('content', content);
    
    // Alcune API potrebbero richiedere POST con metodo _method=PUT o form-data
    // Nel caso di Simple API, vediamo se accetta x-www-form-urlencoded o JSON
    // Qui assumiamo un esempio standard con JSON per semplicità, ma controlla la doc specifica
    
    try {
        const response = await fetch(`${API_URL}/posts/${id}`, {
            method: "PUT",
            headers: {
                 // Spesso con PUT e file siumulati si usa POST + _method, 
                 // ma qui proviamo l'approccio standard se supportato
                "Authorization": `Bearer ${token}`,
                "Content-Type": "application/x-www-form-urlencoded" 
            },
            body: data
        });

        const result = await response.json();
        console.log("Post aggiornato:", result);
    } catch (error) {
        console.error("Errore aggiornamento:", error);
    }
}

Eliminazione (DELETE)

Per eliminare una risorsa, usiamo il metodo DELETE sull'URL specifico della risorsa.

async function deletePost(id) {
    const token = localStorage.getItem("token");
    
    if (!confirm("Sei sicuro di voler eliminare questo post?")) return;

    try {
        const response = await fetch(`${API_URL}/posts/${id}`, {
            method: "DELETE",
            headers: {
                "Authorization": `Bearer ${token}`
            }
        });

        const result = await response.json();
        console.log("Post eliminato:", result);
        
        // Aggiorna la UI rimuovendo l'elemento
        getPosts(); // Ricarica la lista
    } catch (error) {
        console.error("Errore eliminazione:", error);
    }
}

Una alternativa open source a Firebase. Offre un database PostgreSQL completo, autenticazione, API istantanee e funzionalità real-time.
