Introduzione

SASS (Syntactically Awesome Stylesheets) e SCSS sono preprocessori CSS: estendono CSS con strumenti che non esistono nel linguaggio base, poi generano un file .css standard leggibile dal browser.

Idea chiave: il browser non legge direttamente SCSS/SASS. Serve sempre una compilazione verso CSS. 
// input: style.scss
$primary-color: #0d6efd;

.button {
  background: $primary-color;
  &:hover {
    background: darken($primary-color, 8%);
  }
}
/* output: style.css */
.button {
  background: #0d6efd;
}

.button:hover {
  background: #0a58ca;
}

Differenza tra SASS e SCSS

SASS e SCSS usano lo stesso motore, ma una sintassi diversa:

Formato Estensione Sintassi Consiglio didattico
SCSS .scss Molto simile a CSS (usa {} e ;) Scelta migliore per iniziare
SASS .sass Niente parentesi graffe o punto e virgola Meno usata nei team moderni

Esempio SCSS

.card {
  padding: 1rem;
  .title {
    font-size: 1.25rem;
  }
}

Esempio SASS

.card
  padding: 1rem
  .title
    font-size: 1.25rem

Funzionalita Principali

1. Variabili

$spacing-md: 1rem;
$brand: #198754;

.alert-success {
  padding: $spacing-md;
  border-left: 4px solid $brand;
}

2. Nesting

.nav {
  display: flex;

  li {
    list-style: none;
  }

  a {
    text-decoration: none;

    &:hover {
      text-decoration: underline;
    }
  }
}

3. Mixin

@mixin flex-center {
  display: flex;
  justify-content: center;
  align-items: center;
}

.hero {
  @include flex-center;
  min-height: 50vh;
}

4. Partials e Import moderni

// _variables.scss
$primary: #0d6efd;

// main.scss
@use "variables" as v;

.btn-primary {
  background-color: v.$primary;
}
Attenzione: preferisci @use e @forward al vecchio @import, oggi deprecato in Sass.

Struttura File Consigliata

Per progetti didattici e reali, separa i file SCSS per responsabilita:

scss/
  _variables.scss
  _mixins.scss
  _base.scss
  _components.scss
  _layout.scss
  main.scss

In main.scss importi tutto in ordine logico:

@use "variables";
@use "mixins";
@use "base";
@use "layout";
@use "components";

Compilatori CSS

I compilatori trasformano SCSS/SASS in CSS. I piu usati oggi:

Dart Sass (standard ufficiale)

E il compilatore di riferimento mantenuto dal team Sass.

# installazione globale
npm install -g sass

# compilazione singola
sass scss/main.scss css/style.css

# watch mode (consigliato in sviluppo)
sass --watch scss/main.scss:css/style.css

Live Sass Compiler (estensione VS Code)

Compila automaticamente al salvataggio senza configurazioni complesse. Utile in contesti didattici.

Vite / Webpack / Parcel

Nei progetti moderni il compilatore e integrato nel bundler, insieme a autoprefixer, minificazione e gestione moduli.

Best practice: mantieni SCSS in una cartella separata e committa sia i sorgenti SCSS sia il CSS compilato se il progetto e statico.

Workflow Consigliato in Classe

  1. Scrivi componenti e layout in SCSS.
  2. Compila in CSS con watch mode durante lo sviluppo.
  3. Verifica il risultato nel browser e in DevTools.
  4. Ottimizza il CSS finale per la produzione (minify).
# esempio workflow rapido
sass --watch scss/main.scss:css/style.css

Pro e Contro

Vantaggi

  • Codice piu modulare e riusabile
  • Design system coerente grazie alle variabili
  • Riduce duplicazioni con mixin e funzioni
  • Workflow professionale simile ai team reali

Svantaggi

  • Passaggio di compilazione in piu
  • Nesting eccessivo puo complicare il debug
  • Richiede ordine nella struttura dei file
  • Se usato male genera CSS inutile o troppo specifico
