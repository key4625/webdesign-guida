HTML Semantico

L'HTML semantico usa tag che descrivono il significato del contenuto, non solo la presentazione.

Non Semantico
<div id="header">
    <div id="nav">...</div>
</div>
<div id="content">
    <div class="post">...</div>
</div>
<div id="sidebar">...</div>
<div id="footer">...</div>
Semantico
<header>
    <nav>...</nav>
</header>
<main>
    <article>...</article>
</main>
<aside>...</aside>
<footer>...</footer>

Vantaggi dell'HTML Semantico

  • Accessibilità: Gli screen reader comprendono meglio la struttura
  • SEO: I motori di ricerca capiscono meglio il contenuto
  • Manutenibilità: Codice più leggibile e comprensibile
  • Standardizzazione: Segue gli standard web moderni

Tag Semantici Essenziali

Tag Utilizzo Esempio
<header> Intestazione del sito o sezione Logo, navigazione principale
<nav> Menu di navigazione Menu principale, breadcrumb
<main> Contenuto principale (uno per pagina) Contenuto unico della pagina
<article> Contenuto autonomo Post blog, articolo news
<section> Sezione tematica Capitolo, tab panel
<aside> Contenuto correlato Sidebar, box correlati
<footer> Piè di pagina Copyright, link utili

Accessibilità (A11y)

Rendere il web accessibile a tutti, incluse persone con disabilità.

Principi WCAG (Web Content Accessibility Guidelines)

Percepibile

Le informazioni devono essere presentate in modi percepibili dagli utenti.

Operabile

L'interfaccia deve essere navigabile e utilizzabile.

Comprensibile

Contenuto e interfaccia devono essere comprensibili.

Robusto

Compatibile con tecnologie assistive attuali e future.

SEO (Search Engine Optimization)

Ottimizzare il sito per i motori di ricerca:

Meta Tags Essenziali

<head>
    <!-- Titolo (50-60 caratteri) -->
    <title>Guida HTML e CSS - Web Design per Studenti</title>
    
    <!-- Descrizione (150-160 caratteri) -->
    <meta name="description" content="Guida completa e pratica di HTML e CSS per studenti di web design. Esempi, best practices e codice pronto all'uso.">
    
    <!-- Charset -->
    <meta charset="UTF-8">
    
    <!-- Viewport per mobile -->
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    
    <!-- Keywords (meno importante ora) -->
    <meta name="keywords" content="html, css, web design, guida, tutorial">
    
    <!-- Author -->
    <meta name="author" content="Nome Autore">
    
    <!-- Open Graph per social media -->
    <meta property="og:title" content="Guida HTML e CSS">
    <meta property="og:description" content="Guida completa per studenti">
    <meta property="og:image" content="https://esempio.it/immagine.jpg">
    <meta property="og:url" content="https://esempio.it">
    <meta property="og:type" content="website">
    
    <!-- Twitter Card -->
    <meta name="twitter:card" content="summary_large_image">
    <meta name="twitter:title" content="Guida HTML e CSS">
    <meta name="twitter:description" content="Guida completa">
    <meta name="twitter:image" content="https://esempio.it/immagine.jpg">
    
    <!-- Favicon -->
    <link rel="icon" type="image/png" href="favicon.png">
    
    <!-- Canonical URL -->
    <link rel="canonical" href="https://esempio.it/pagina">
</head>

Best Practices SEO

  • URL Descrittive
    ✅ esempio.it/guida-html-css
    ❌ esempio.it/pagina123.php?id=456
  • Struttura Heading Gerarchica
    Un solo <h1> per pagina, seguire ordine logico
  • Contenuto di Qualità
    Originale, utile, ben scritto, aggiornato
  • Link Interni
    Collegare pagine correlate del sito
  • Velocità di Caricamento
    Ottimizzare immagini, minimizzare CSS/JS
  • Mobile-Friendly
    Responsive design è essenziale
  • HTTPS
    Certificato SSL per sicurezza
  • Sitemap.xml
    Mappa del sito per i crawler

Performance e Ottimizzazione

Ottimizzazione Immagini

<!-- Formati moderni con fallback -->
<picture>
    <source srcset="immagine.webp" type="image/webp">
    <source srcset="immagine.jpg" type="image/jpeg">
    <img src="immagine.jpg" alt="Descrizione" loading="lazy">
</picture>

<!-- Lazy loading -->
<img src="foto.jpg" alt="Foto" loading="lazy">

<!-- Responsive images -->
<img srcset="small.jpg 480w, medium.jpg 800w, large.jpg 1200w"
     sizes="(max-width: 600px) 480px, (max-width: 900px) 800px, 1200px"
     src="medium.jpg" alt="Descrizione">

Minimizzazione e Compressione

  • Minimizza CSS e JavaScript
    Rimuovi spazi, commenti e caratteri non necessari
  • Comprimi immagini
    Usa strumenti come TinyPNG, ImageOptim
  • Abilita compressione Gzip/Brotli
    Sul server web
  • Usa CDN
    Content Delivery Network per librerie comuni

Caricamento Risorse

<!-- CSS nel head -->
<link rel="stylesheet" href="style.css">

<!-- Preconnect a domini esterni -->
<link rel="preconnect" href="https://fonts.googleapis.com">

<!-- JavaScript alla fine del body o con defer -->
<script src="script.js" defer></script>
<script src="script.js" async></script>

<!-- Preload risorse critiche -->
<link rel="preload" href="font.woff2" as="font" type="font/woff2" crossorigin>
Strumenti di Analisi:
  • Google PageSpeed Insights
  • Lighthouse (DevTools Chrome)
  • WebPageTest
  • GTmetrix

Responsive Design Best Practices

Approccio Mobile-First

/* Stili base per mobile */
.container {
    width: 100%;
    padding: 15px;
}

.menu {
    flex-direction: column;
}

/* Tablet e oltre */
@media (min-width: 768px) {
    .container {
        width: 750px;
        margin: 0 auto;
    }
    
    .menu {
        flex-direction: row;
    }
}

/* Desktop */
@media (min-width: 1200px) {
    .container {
        width: 1140px;
    }
}

Unità Relative

/* Usa unità relative invece di px quando possibile */
.elemento {
    font-size: 1rem;      /* Relativo al root */
    padding: 1em;         /* Relativo al font-size dell'elemento */
    width: 80%;           /* Relativo al parent */
    max-width: 1200px;    /* Limite massimo in px */
    margin: 2rem auto;
}

/* Viewport units */
.hero {
    height: 100vh;        /* 100% altezza viewport */
    width: 100vw;         /* 100% larghezza viewport */
}

.box {
    width: 50vw;          /* 50% larghezza viewport */
    font-size: 2vw;       /* 2% larghezza viewport */
}

Immagini Responsive

img {
    max-width: 100%;
    height: auto;
    display: block;
}

/* Classe utility Bootstrap */
.img-fluid {
    max-width: 100%;
    height: auto;
}

Convenzioni di Nomenclatura

Metodologia BEM (Block Element Modifier)

<!-- Block -->
<div class="card">
    <!-- Element -->
    <div class="card__header">
        <h2 class="card__title">Titolo</h2>
    </div>
    <div class="card__body">
        <p class="card__text">Testo...</p>
        <!-- Modifier -->
        <button class="card__button card__button--primary">Azione</button>
    </div>
</div>
/* Block */
.card { }

/* Element */
.card__header { }
.card__title { }
.card__body { }
.card__button { }

/* Modifier */
.card--featured { }
.card__button--primary { }
.card__button--disabled { }

Altre Convenzioni

  • Nomi descrittivi
    ✅ .user-profile-avatar
    ❌ .img1
  • Kebab-case per CSS
    .main-navigation, .sidebar-widget
  • camelCase per JavaScript
    getElementById, userName
  • File e cartelle lowercase
    index.html, style.css, images/

Sicurezza di Base

Validazione Input

<!-- Usa attributi HTML5 per validazione -->
<input type="email" required>
<input type="url" required>
<input type="number" min="0" max="100">
<input type="text" pattern="[A-Za-z]{3,}" required>

Link Esterni Sicuri

<!-- Usa rel="noopener" per link esterni con target="_blank" -->
<a href="https://esempio.it" target="_blank" rel="noopener noreferrer">
    Link esterno
</a>

Content Security Policy

<meta http-equiv="Content-Security-Policy" 
      content="default-src 'self'; script-src 'self' https://cdn.jsdelivr.net">
Importante:
  • Non inserire mai dati sensibili nel codice client-side
  • Usa HTTPS per tutti i siti in produzione
  • Mantieni aggiornate le dipendenze esterne
  • Valida sempre gli input utente
