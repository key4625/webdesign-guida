Come funziona il nostro Blog con API e JS

Obiettivo Didattico

In questo progetto stiamo trasformando un sito statico in una Applicazione Web Dinamica. I dati (gli articoli) non sono scritti fissi nell'HTML, ma vengono scaricati da un server esterno tramite API.

Per mantenere il codice ordinato, abbiamo diviso il sito in due macro-aree:

Area Pubblica
  • index.html: La Home page accessibile a tutti.
  • js/ins-content.js: Lo script che scarica e mostra gli articoli.
Qui l'utente può solo LEGGERE (Richieste GET).
Area Admin (Privata)
  • admin/: Cartella che contiene login e dashboard.
  • js/auth.js: Gestisce il Login e il Token.
Qui l'amministratore può SCRIVERE (POST, PUT, DELETE).

Come facciamo a sapere se un utente ha il permesso di cancellare un articolo? Usiamo un sistema basato su Token (una sorta di pass digitale).

  1. L'utente inserisce email e password nel form di Login.
  2. Il JavaScript invia questi dati all'API (POST /login).
  3. Se i dati sono corretti, l'API risponde con un Token.
  4. Noi salviamo questo token nel browser usando il localStorage.
    localStorage.setItem('authToken', token)
Sicurezza: Le pagine protette (come la Dashboard) controllano subito se questo token esiste. Se non c'è, ti rispediscono al Login!

Per interagire con il database, usiamo diversi "verbi" HTTP nella funzione fetch():

AzioneMetodoServe il Token?
Leggere (Home)GETNo
Creare nuovoPOSTSì
ModificarePUTSì
CancellareDELETESì

Esempio di codice per creare un articolo:

fetch(apiUrl + 'posts', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer ' + localStorage.getItem('authToken') // Alleghiamo il Token!
  },
  body: JSON.stringify(datiArticolo)
})

Guida Dettagliata ai File

Ecco cosa fa esattamente ogni file che abbiamo creato:

Frontend (Pubblico)

index.html

La pagina principale visibile a tutti.

  • Usa <div id="listaArticoli"> come contenitore vuoto.
  • Chiama lo script ins-content.js per riempirsi.
js/ins-content.js

Il "motore" che scarica gli articoli.

  • Fase 1: fetch(url) chiede i dati all'API (GET).
  • Fase 2: Converte la risposta in JSON.
  • Fase 3: Usa document.createElement per creare HTML dinamico.

Backend (Admin)

admin/index.html

La pagina di Login.

  • Contiene un Form HTML semplice.
  • Intercetta l'invio con JS per evitare il ricaricamento della pagina (preventDefault).
js/auth.js

Gestione Sicurezza e Token.

  • Invia Email/Password all'API (POST).
  • Se OK, riceve un Token.
  • Salva il Token nel browser usando localStorage.setItem().
  • Reindirizza alla Dashboard.
