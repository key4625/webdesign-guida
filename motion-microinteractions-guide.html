Motion e Micro-interazioni

Animazioni utili, non decorative: feedback, orientamento e chiarezza

Scheda Didattica
Obiettivo: progettare motion funzionale che migliori chiarezza e feedback UI.
Prerequisiti: CSS transition/animation base.
Durata stimata: 50 min
Output: set minimo di micro-interazioni accessibili.
Indice
1. Perche usare motion 2. Tipi di micro-interazione 3. Timing ed easing 4. Pattern pratici 5. Accessibilita motion

1. Perche usare motion

La motion guida l'attenzione e chiarisce cosa sta succedendo nell'interfaccia: click riuscito, caricamento, cambio stato, errore.

Obiettivo: ridurre incertezza utente, non aggiungere rumore visivo.

2. Tipi di micro-interazione

  • Hover e focus visibili su bottoni/link.
  • Feedback submit form (loading, success, error).
  • Expand/collapse di sezioni (accordion).
  • Transizioni tra pagine/sezioni.

3. Timing ed easing

Range utili:

  • Hover: 120-180ms
  • UI transizioni semplici: 200-300ms
  • Entrata elementi: 300-450ms
.btn {
  transition: transform 180ms ease, box-shadow 180ms ease;
}

.btn:hover {
  transform: translateY(-2px);
}

4. Pattern pratici

Stagger reveal cards

.card {
  opacity: 0;
  transform: translateY(12px);
  animation: reveal 420ms ease forwards;
}

@keyframes reveal {
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

Feedback stato bottone

Cambia testo/icona durante il caricamento e mostra chiaramente l'esito.

5. Accessibilita motion

Rispetta utenti sensibili al movimento usando la media query dedicata:

@media (prefers-reduced-motion: reduce) {
  * {
    animation: none !important;
    transition: none !important;
    scroll-behavior: auto !important;
  }
}
