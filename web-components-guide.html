Introduzione ai Web Components

I Web Components sono una suite di tecnologie native del browser che permettono di creare nuovi tag HTML personalizzati, incapsulati e riutilizzabili.

A differenza di framework come React o Vue, i Web Components funzionano nativamente nel browser senza bisogno di build tools o librerie esterne.

Immagina questo: Invece di copiare e incollare il codice della navbar in ogni pagina HTML (<nav class="...">...</nav>), puoi semplicemente scrivere <my-navbar></my-navbar>!

Perché usarli per il Layout?

Uno dei problemi classici nello sviluppo di siti statici multipagina è la gestione delle parti comuni (Header, Footer, Sidebar). Se devi cambiare una voce del menu, devi modificare 10, 20 o 50 file HTML.

I Web Components risolvono questo problema centralizzando il codice in un unico file JavaScript.

  • Centralizzazione: Modifichi il componente in un punto, si aggiorna ovunque.
  • Pulizia: Il tuo file HTML contiene solo il contenuto specifico della pagina.
  • Nessun Backend: Funziona anche se apri il file HTML direttamente dal desktop (doppio click), senza bisogno di un server PHP o Node.js.

Creare un Header Personalizzato

Creiamo un componente <site-header>. Per farlo, dobbiamo estendere la classe HTMLElement di JavaScript.

1. Il Codice JavaScript (es. components.js)

class SiteHeader extends HTMLElement {
    // Il metodo connectedCallback viene chiamato quando l'elemento viene aggiunto alla pagina
    connectedCallback() {
        this.innerHTML = `
            <header style="background: #333; color: white; padding: 1rem;">
                <nav>
                    <a href="index.html" style="color: white; margin-right: 15px;">Home</a>
                    <a href="chi-siamo.html" style="color: white; margin-right: 15px;">Chi Siamo</a>
                    <a href="contatti.html" style="color: white;">Contatti</a>
                </nav>
            </header>
        `;
    }
}

// 2. Registrazione del componente (il nome deve contenere un trattino "-")
customElements.define('site-header', SiteHeader);

2. Utilizzo nell'HTML

<!DOCTYPE html>
<html>
<head>
    <script src="js/components.js" defer></script>
</head>
<body>
    
    <!-- Ecco il nostro componente personalizzato! -->
    <site-header></site-header>

    <main>
        <h1>Benvenuti</h1>
        <p>Contenuto della pagina...</p>
    </main>

</body>
</html>

Gestire la Voce di Menu Attiva

Un problema comune è evidenziare la pagina corrente nel menu. Possiamo risolvere passando un attributo al nostro componente.

Nell'HTML

Indichiamo quale pagina siamo usando un attributo personalizzato, ad esempio active-page.

<!-- Sulla Home Page -->
<site-header active-page="home"></site-header>

<!-- Sulla pagina Chi Siamo -->
<site-header active-page="chi-siamo"></site-header>

Nel JavaScript

Leggiamo l'attributo e aggiungiamo una classe CSS (es. active) al link corretto.

class SiteHeader extends HTMLElement {
    connectedCallback() {
        // Leggiamo l'attributo 'active-page'
        const activePage = this.getAttribute('active-page');

        this.innerHTML = `
            <header>
                <nav>
                    <a href="index.html" class="${activePage === 'home' ? 'active' : ''}">Home</a>
                    <a href="chi-siamo.html" class="${activePage === 'chi-siamo' ? 'active' : ''}">Chi Siamo</a>
                    <a href="contatti.html" class="${activePage === 'contatti' ? 'active' : ''}">Contatti</a>
                </nav>
            </header>
        `;
    }
}
customElements.define('site-header', SiteHeader);
Nota: Per stili più complessi o isolati, potresti voler utilizzare lo Shadow DOM (this.attachShadow({mode: 'open'})), ma per header globali spesso il DOM normale è più semplice perché eredita gli stili CSS (come Bootstrap) della pagina principale.

Alternative e Confronto

I Web Components non sono l'unico modo per includere parti comuni. Ecco un confronto con altre tecniche popolari:

Se hai un server web (Apache, Nginx), questa è spesso la soluzione più robusta.

<?php include 'header.php'; ?>
  • Pro: L'HTML arriva al browser già completo (ottimo per la SEO).
  • Contro: Richiede un server funzionante (niente doppio click sul file).

Puoi caricare un file HTML esterno e "iniettarlo" in un div vuoto.

fetch('header.html')
  .then(response => response.text())
  .then(data => {
      document.getElementById('header-container').innerHTML = data;
  });
  • Pro: Semplice, tiene il codice HTML separato.
  • Contro: Può causare un leggero "sfarfallio" all'avvio; difficile evidenziare la voce di menu attiva (devi manipolare il DOM dopo il caricamento).
