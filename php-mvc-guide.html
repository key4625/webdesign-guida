Introduzione

MVC e un pattern architetturale che separa dati, logica e interfaccia. In PHP e fondamentale quando il progetto cresce oltre pagine sparse con codice misto.

Obiettivo: evitare file "tutto in uno" difficili da mantenere.

Cos'e MVC

  • Model: rappresenta i dati e dialoga con il database.
  • View: mostra HTML all'utente.
  • Controller: riceve la richiesta, coordina model e view.
Browser -> Router -> Controller -> Model -> Controller -> View -> Browser

Ruoli di Model, View e Controller

Layer Responsabilita Non dovrebbe fare
Model Query DB, regole dati, validazioni dominio Stampare HTML
View Render HTML e output Fare query SQL
Controller Gestire request/response e orchestrare flusso Contenere logica SQL estesa

Flusso di una Richiesta

  1. L'utente visita /studenti.
  2. Il router invia la richiesta a StudenteController@index.
  3. Il controller chiede i dati al model Studente.
  4. Il controller passa i dati alla view.
  5. La view genera HTML e lo invia al browser.

Struttura Cartelle Consigliata

progetto-php/
  public/
    index.php
  app/
    Controllers/
      StudenteController.php
    Models/
      Studente.php
    Views/
      studenti-index.php
  routes/
    web.php
Nota: la cartella public contiene il punto di ingresso, il resto del codice rimane separato e ordinato.

Mini Esempio Completo

Model

<?php
class Studente {
    public static function all(): array {
        return [
            ["nome" => "Anna"],
            ["nome" => "Marco"],
            ["nome" => "Luca"]
        ];
    }
}
?>

Controller

<?php
class StudenteController {
    public function index(): void {
        $studenti = Studente::all();
        require __DIR__ . '/../Views/studenti-index.php';
    }
}
?>

View

<h2>Elenco Studenti</h2>
<ul>
  <?php foreach ($studenti as $s): ?>
    <li><?= htmlspecialchars($s['nome']) ?></li>
  <?php endforeach; ?>
</ul>

Routing Base

Un router minimale puo basarsi su path della URL:

<?php
$path = parse_url($_SERVER['REQUEST_URI'], PHP_URL_PATH);

if ($path === '/studenti') {
    (new StudenteController())->index();
} else {
    http_response_code(404);
    echo 'Pagina non trovata';
}
?>

Nei framework (Laravel, Symfony) il routing e molto piu potente ma il principio resta lo stesso.

Pro e Contro di MVC

Vantaggi

  • Codice piu ordinato e leggibile
  • Facile manutenzione in team
  • Separazione netta delle responsabilita
  • Scalabilita migliore nei progetti medi/grandi

Svantaggi

  • Setup iniziale piu lungo
  • Piu file da gestire rispetto a script semplici
  • Serve disciplina architetturale

Confronto con JavaScript
