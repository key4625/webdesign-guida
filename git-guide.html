Guida Git e GitHub Pages

Versionare il codice, lavorare meglio in team e pubblicare i progetti online

1. Cos'e Git

Git e un sistema di controllo versione. In pratica, salva la storia del tuo progetto nel tempo.

Ogni volta che fai un cambiamento importante, puoi creare uno "snapshot" del codice. Se qualcosa si rompe, puoi tornare indietro in modo sicuro.

Pensa a Git come a una macchina del tempo per i tuoi file.

2. Perche e utile nei progetti

  • Ti permette di lavorare senza paura di perdere versioni funzionanti.
  • Aiuta il lavoro di gruppo: ognuno puo lavorare su parti diverse.
  • Rende chiaro cosa e cambiato e quando.
  • Si integra con GitHub per avere backup online e collaborazione.
  • Ti permette di pubblicare facilmente progetti statici con GitHub Pages.

3. Workflow base: add, commit, push

Il ciclo minimo che userai piu spesso e questo:

git add .
git commit -m "Messaggio chiaro del cambiamento"
git push origin master

git add: prepara i file modificati.

git commit: salva localmente una versione con descrizione.

git push: invia i commit su GitHub.

4. Avviare il primo repository

Se parti da un progetto locale, questi sono i passaggi base:

git init
git add .
git commit -m "Primo commit"
git branch -M master
git remote add origin https://github.com/USERNAME/NOME-REPO.git
git push -u origin master

Dopo il primo push, il tuo progetto e online su GitHub.

5. Pubblicare con GitHub Pages

GitHub Pages ti permette di pubblicare un sito statico direttamente da un repository.

  1. Rendi il repository pubblico su GitHub.
  2. Vai in Settings > Pages.
  3. In Build and deployment scegli Deploy from a branch.
  4. Seleziona branch master (o main) e cartella root.
  5. Salva e attendi 1-3 minuti.

URL tipico di pubblicazione:

https://username.github.io/nome-repository/
Ottimo per portfolio, progetti di corso, esercitazioni e demo per colloqui.

6. Consigli pratici per studenti

  • Fai commit piccoli e frequenti, non uno solo a fine giornata.
  • Scrivi messaggi commit chiari: cosa hai fatto e perche.
  • Prima di iniziare una nuova lezione, fai pull per allinearti.
  • Usa GitHub Pages per mostrare i progetti senza bisogno di server.
