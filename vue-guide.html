Teoria Essenziale

Capire la Struttura

Perché Vue.js?

Immagina di costruire una casa Lego senza istruzioni e mischiando tutti i pezzi in un unico scatolone. Ecco, quello è il "JavaScript Spaghetti Code".

Vue.js è come un organizzatore: ti costringe a dividere i pezzi per colore e funzione, mettendoli in scatolette ordinate (i Componenti). In questo modo, se devi cambiare il tetto, sai esattamente dove trovare i pezzi senza rompere le fondamenta.

Vantaggio: Codice pulito, riutilizzabile e facile da leggere.

L'Anatomia di un Componente (.vue)

Ogni file Vue è un piccolo mondo autonomo diviso in tre piani:

<template>
La Struttura (HTML)
<script>
Il Cervello (Logic)
<style>
Il Vestito (CSS)
App.vue 
<template>
  <!-- Qui va l'HTML -->
  <div class="card">
    <h1>{{ titolo }}</h1>
    <button @click="saluta">Cliccami</button>
  </div>
</template>

<script>
export default {
  // 1. DATA: La memoria del componente
  data() {
    return {
      titolo: "Ciao classe!"
    }
  },
  // 2. METHODS: Le azioni che può compiere
  methods: {
    saluta() {
      alert("Benvenuto in Vue.js!");
    }
  }
}
</script>

<style>
/* Qui va il CSS solo per questo componente */
h1 { color: blue; }
</style>

Le "Armi" principali (Direttive)

Le direttive sono istruzioni speciali che diamo all'HTML per renderlo intelligente. Iniziano tutte con v-.

{{ Mustache }}
Interpolazione

Stampa il valore di una variabile direttamente nell'HTML.

<p>Ciao {{ nomeUtente }}</p>
v-if / v-else
Condizioni

Decide se mostrare o nascondere un elemento.

<p v-if="loggato">Benvenuto</p>
<p v-else>Accedi</p>
v-for
Cicli

Ripete un elemento per ogni oggetto in una lista.

<li v-for="alunno in classe">
  {{ alunno }}
</li>
@click (v-on)
Eventi

Ascolta eventi come il click del mouse.

<button @click="faiQualcosa">
  Invia
</button>

Laboratorio

Progetto: Il Sito della Classe

Creiamo un mini-sito passo dopo passo per capire come i pezzi si uniscono.

Step 1 Ordine e Componenti

Obiettivo: Pulire e organizzare. Invece di scrivere tutto in un solo file, creiamo dei pezzi Lego.

Struttura file da creare:

  • components/Header.vue (Contiene il titolo)
  • components/Footer.vue (Contiene il copyright)

Come usarli in App.vue:

<script>
// 1. IMPORTARE
import HeaderClasse from './components/Header.vue'
import FooterClasse from './components/Footer.vue'

export default {
  // 2. REGISTRARE
  components: {
    HeaderClasse,
    FooterClasse
  }
}
</script>

<template>
  <div id="app">
    <!-- 3. USARE -->
    <HeaderClasse />
    <main>Contenuto principale...</main>
    <FooterClasse />
  </div>
</template>
Concetto Chiave (Modularità): "Divide et Impera". Dividere il problema grande in problemi piccoli risolvibili separatamente.

Step 2 Comunicazione (Props)

Obiettivo: Rendere i componenti dinamici. Vogliamo che l'Header scriva il nome della classe, ma il nome glielo deve passare l'App principale.

Padre (App.vue)
<template>
   <!-- Passaggio dati verso il basso -->
   <HeaderClasse titolo="Classe 5^A" />
</template>
Figlio (Header.vue)
<script>
export default {
  // Il figlio dichiara cosa si aspetta
  props: ['titolo']
}
</script>

<template>
  <h1>Benvenuti nella {{ titolo }}</h1>
</template>

Step 3 Reattività (Data Binding)

Obiettivo: La magia di Vue. Creiamo un contatore di "Mi Piace". Quando il dato cambia, lo schermo si aggiorna da solo.

<template>
  <div class="like-box">
    <h3>Ti piace questo sito?</h3>
    <p>Numero likes: {{ likes }}</p>
    
    <!-- Al click, incrementa la variabile -->
    <button @click="likes++" class="btn btn-primary">
      👍 Mi Piace
    </button>
  </div>
</template>

<script>
export default {
  data() {
    return {
      likes: 0 // Stato iniziale
    }
  }
}
</script>

Step 4 Navigazione (Cenni di Routing)

Obiettivo: Simulare un sito multipagina senza ricaricare.

In Vue non si usa <a href="..."> perché ricaricherebbe tutta la pagina. Si usa il Vue Router.

Immagina il Router come un Portiere d'albergo. Invece di farti uscire dall'albergo e rientrare per cambiare stanza (ricaricare la pagina), il portiere ti accompagna semplicemente alla porta giusta.

  • <router-link to="/">: È il link magico (sostituisce a href).
  • <router-view>: È la "cornice" dove Vue mostrerà il componente della pagina (Home o Chi Siamo) a seconda dell'indirizzo.

Step 5 Dati dal Mondo Reale (API & Fetch)

Obiettivo: Connettere il sito a un CMS esterno (come Strapi, WordPress) o a un servizio dati.

Finora abbiamo scritto i dati a mano nel codice (hardcoded). Nel mondo reale, i dati arrivano da un Server tramite API (Application Programming Interface).

Il concetto chiave: Lifecycle Hooks

Quando "nasce" un componente? Quando viene visualizzato a schermo. Vue ci permette di eseguire codice esattamente in quel momento tramite la funzione mounted().

È come dire: "Appena questa pagina è pronta, telefona al server e fatti dare gli articoli del blog".

Flusso tipico:
1. Pagina vuota (loading...)
2. mounted() chiama l'API
3. I dati arrivano
4. Vue aggiorna la pagina automaticamente
Esempio Pratico (Lista Utenti)
<script>
export default {
  data() {
    return {
      utenti: [],    // 1. Array vuoto all'inizio
      caricamento: true
    }
  },
  // 2. Appena il componente è montato...
  async mounted() {
    try {
      // Simuliamo una chiamata a Strapi o un'API pubblica
      const risposta = await fetch('https://jsonplaceholder.typicode.com/users');
      const dati = await risposta.json();
      
      this.utenti = dati; // 3. Riempiamo l'array con i dati veri
    } catch (errore) {
      console.error("Ops, errore nel server:", errore);
    } finally {
      this.caricamento = false; // Stop rotellina di caricamento
    }
  }
}
</script>

<template>
  <div>
    <h2>I nostri Utenti</h2>
    
    <!-- Messaggio mentre carica -->
    <p v-if="caricamento">Caricamento in corso...</p>
    
    <!-- Lista finale -->
    <ul v-else>
      <li v-for="utente in utenti" :key="utente.id">
        {{ utente.name }} ({{ utente.email }})
      </li>
    </ul>
  </div>
</template>
