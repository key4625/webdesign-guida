Design System

Cos'e, cosa comprende e come costruirlo in un progetto reale

Scheda Didattica
Obiettivo: capire struttura e logica di un design system applicabile a progetti reali.
Prerequisiti: nozioni base di UI design e CSS variables.
Durata stimata: 75 min
Output: schema iniziale di token + componenti core del progetto.
Indice
1. Definizione 2. Cosa comprende 3. Design token 4. Libreria componenti 5. Processo di sviluppo 6. Documentazione

1. Definizione

Un design system e un insieme coordinato di regole, componenti e principi che permette a team di progettare e sviluppare interfacce coerenti nel tempo.

Non e solo una libreria UI: include anche linguaggio, comportamento e standard di qualità.

2. Cosa comprende

  • Fondazioni visive: colori, tipografia, spacing, griglie.
  • Componenti: button, card, form, modal, nav, alert.
  • Pattern: composizioni tipiche (hero, lista, dashboard).
  • Linee guida: voce del brand, microcopy, stati e interazioni.
  • Accessibilita: contrasti, focus, tastiera, semantica.

3. Design token

I token sono variabili astratte che collegano design e codice.

:root {
  --color-primary: #2563eb;
  --color-surface: #ffffff;
  --text-default: #0f172a;
  --space-1: 4px;
  --space-2: 8px;
  --radius-md: 10px;
}

Con i token puoi aggiornare un intero sistema senza riscrivere ogni componente.

4. Libreria componenti

Schema consigliato per ogni componente

  1. Nome e scopo.
  2. Varianti (primary, secondary, ghost).
  3. Stati (default, hover, active, disabled, error).
  4. Regole di utilizzo corretto/errato.
<button class="btn btn-primary">Conferma</button>
<button class="btn btn-outline-primary">Annulla</button>

5. Processo di sviluppo

  1. Audit dell'interfaccia esistente.
  2. Definizione delle fondazioni visive.
  3. Costruzione componenti base.
  4. Composizione in pattern e template pagina.
  5. Versionamento e governance.
Parti piccolo: 6-8 componenti core ben fatti valgono piu di una libreria enorme incoerente.

6. Documentazione efficace

Ogni componente deve avere una scheda sintetica con esempi, regole e snippet riutilizzabili.

  • Quando usare il componente.
  • Quando NON usarlo.
  • Snippet base in HTML/CSS.
  • Vincoli responsive e accessibilita.
